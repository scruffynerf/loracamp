# LoraCamp Retheming Guide

This guide explains how to customize the visual appearance of your LoraCamp site. LoraCamp uses a flexible, variable-based CSS system that allows for everything from simple color tweaks to complete structural overhauls—all without touching the HTML templates.

## Theming Architecture

LoraCamp uses a **CSS Variables (Custom Properties)** based theming system. The visual appearance is determined by a set of variables defined on the `:root` element.

### Dynamic Theme Generation

Themes are generated by the `loracamp/theming.py` module based on the `[theme]` section in the `catalog.toml` manifest. The Python engine calculates a full palette (Backgrounds, Foregrounds, Accents) from these values and injects them into the site's `<head>`.

### Theme Switching Logic

1. **System Preference**: By default, the site respects `prefers-color-scheme`.
2. **Explicit Toggle**: A JavaScript toggle allows users to force "light" or "dark" mode.
3. **Persistence**: The chosen theme is saved in `localStorage.loracamp_theme`.
4. **Attribute Binding**: The theme is applied by setting the `data-theme` attribute on the `<html>` element.

## Strategy 1: User Configuration (Simple)

The easiest way to change the site's look is by modifying the `[theme]` section in your `catalog.toml` file. LoraCamp uses a functional color system that derives a full, harmonious palette from just a few parameters.

### Configuration Sample

```toml
[theme]
base = "dark"         # Sets initial mode: "dark" or "light"
accent_hue = 210      # HSL hue (0-360). 210 is a classic blue.
accent_chroma = 80    # Saturation intensity (0-100). 80 is very vibrant.
round_corners = true  # Set to true for soft UI, false for sharp/industrial.
```

### How it Works

* **Base Variables**: LoraCamp calculates neutral backgrounds and text based on whether `base` is light or dark.
* **Accent Calculation**: Every interactive element (buttons, progress bars, hover states) is tinted using the `accent_hue`.
* **Cohesion**: The system ensures text contrast is always sufficient against the backgrounds it generates.

## Strategy 2: CSS Overrides & Replacement (Flexible)

If you need more control than the configuration allows, you can provide custom CSS. LoraCamp automatically looks for a file named `custom.css` in your catalog directory. If found, it is copied to the build and linked on every page **after** the internal styles, giving your rules priority.

### How to use `custom.css`

1. Create a file named `custom.css` in your catalog root.
2. **To Override**: Redefine specific variables or rules.
3. **To Replace**: Add rules for all core classes (e.g., `.track`, `.model-card`) to create a completely new look.

### Example Override

```css
:root {
  --bg-acc: #ff00ff; /* Force a neon magenta accent */
  --cover-border-radius: 20px; /* Oversized rounded corners */
}

/* You can also add entirely new rules */
.site-title {
    font-weight: 900;
    text-transform: uppercase;
    letter-spacing: 0.1em;
}
```

## Strategy 3: Python Plugins (Advanced)

For developers who want to add entirely new named palettes or complex logic without modifying the core LoraCamp code, you can use the **Plugin System**.

LoraCamp automatically loads any `.py` files found in the `plugins/` directory at the project root. This directory is `.gitignore`d by default, making it safe for your custom extensions.

### How to Create a Theme Plugin

1. Create a file like `plugins/my_theme.py`.
2. Define your palette function and a `register(engine)` hook.

```python
from loracamp.theming import register_palette

def my_custom_palette(hue: int, chroma: int):
    return {
        "--bg-1": "#1a1a1a",
        "--fg-1": "#ffffff",
        "--bg-acc": "#00ff00",
        "--fg-acc": "#000000",
        # ... see the variable reference below
    }

def register(engine):
    # This hook is called automatically during the build
    register_palette("neon", my_custom_palette)
```

1. Use your new theme in `catalog.toml`:

```toml
[theme]
base = "neon"
```

## Technical CSS Variable Reference

These variables are defined in `theming.py` and used throughout `site.css`.

### Backgrounds (`--bg-*`)

| Variable | Description | Default (Dark) |
| :--- | :--- | :--- |
| `--bg-1` | Primary page background | Dark charcoal |
| `--bg-1-90` | Translucent background for blur effects | `hsla` version of `--bg-1` |
| `--bg-1-overlay` | Slightly opaque overlay | `hsla` version of `--bg-1` |
| `--bg-2` | Secondary background (cards, buttons) | Slightly lighter gray |
| `--bg-3` | Tertiary background (borders, dividers) | Medium gray |
| `--bg-mg` | Middle-ground gray | Medium-light gray |
| `--bg-acc` | Accent background (primary buttons) | Derived from `accent_hue` |
| `--bg-acc-overlay` | Translucent accent overlay | `hsla` version of `--bg-acc` |

### Foregrounds (`--fg-*`)

| Variable | Description | Default (Dark) |
| :--- | :--- | :--- |
| `--fg-1` | Primary text (headings) | Near white |
| `--fg-1-focus` | Hover/Focus state for primary links | pure white |
| `--fg-1-veil` | Subtle container background | Extremely light translucent white |
| `--fg-2` | Secondary text (body) | Light gray |
| `--fg-3` | Tertiary text (metadata, labels) | Medium gray |
| `--fg-3-focus` | Hover state for tertiary text | Lighter gray |
| `--fg-acc` | Text color on accent backgrounds | High-contrast white |

### Utilities

| Variable | Description |
| :--- | :--- |
| `--cover-border-radius` | Controlled by `round_corners` (default `0.4rem` or `0`) |
| `--ul-list-style-type` | Controls list markers (default `disc`) |

## Core Layout & Components Reference

### Layout Structure

LoraCamp uses a modular "Page" system for layout. Modifying these classes in `site.css` (or overriding them in `custom.css`) will affect all pages.

* **Base Grid (`.layout`)**: overall page structure is a 3-row grid: `header`, `main`, `footer`.
* **Page Containers**:
  * `.page_grid`: A centered container for the primary content.
  * `.page_split`: Used for side-by-side layouts (e.g., Creator info vs. Model list).
  * `.page_center`: Fully centered layout.

### Key Class Selectors

* **Navigation**: `.header-container`, `.browse-btn`, `.back-link`.
* **Content**: `.model-card`, `.track`, `.track_playback`, `.gallery`.
* **Audio Player**: `.docked_player`, `.timeline`, `.elements`, `.track_info`.
* **Search Browser**: `#browser`, `.browser-content`, `#results`.
